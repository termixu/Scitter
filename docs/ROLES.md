# 👨‍🏫 Наставник (преподаватель) — архитектор проекта и тимлид

**Зона дел:** общая архитектура игры, написание базового движка, настройка Git и GitHub, код-ревью, слияние веток, помощь с кодом, контроль дедлайнов, координация команды

**Зона репозитория:** main ветка, index.html, style.css, game.js (скелет), README.md, GLOSSARY.md, CODESTYLE.md, .github/pull_request_template.md

**Инструменты:** VS Code, GitHub Desktop, Git, Chrome DevTools, Figma (для схем), Canvas API

**С кем взаимодействует:** Все члены команды (координирует, помогает, проверяет)

**Примеры задач:**

- Создать базовый шаблон index.html, style.css, game.js
- Настроить репозиторий и GitHub Pages
- Расписать задачи на спринт в Issues
- Провести код-ревью всех Pull Request'ов
- Разрешать конфликты при слиянии веток
- Помогать с отладкой, если ребёнок застрял
- Писать сниппеты и примеры кода для команды
- Следить за дедлайнами и собирать ретроспективу

**Результат за спринт:** полностью работающая игра, собранная из кода всех участников, выложенная на GitHub Pages

**Сложность:** абсолютная

# 🧲 Мурад — инженер физики

**Зона дел:** движение мыши, прыжки, гравитация, столкновения со стенами и предметами

**Зона репозитория:** game.js, physics.js

**Инструменты:** VS Code, JavaScript, Canvas, отладчик браузера

**С кем взаимодействует:** Денис (ловушки влияют на движение), Иван (карты определяют стены и платформы)

**Примеры задач:**

- Написать функцию moveMouse() с поддержкой WASD и стрелок
- Добавить гравитацию и прыжок по пробелу
- Настроить столкновения со стенами (чтобы мышь не проходила сквозь них)
- Сделать платформы, по которым можно прыгать сверху
- Добавить эффект скольжения на масляных лужах (совместно с Денисом)

**Результат за спринт:** мышь бегает, прыгает, не проваливается сквозь пол и не проходит сквозь стены

**Сложность:** высокая
