# 📏 CODESTYLE.md — Как мы пишем код в проекте Skitter

Единый стиль кода нужен, чтобы:
- Код был понятен всем участникам команды
- Не возникало споров о том, где ставить пробелы
- Новые участники быстро влились в проект

**Главное правило:** пиши так, будто через месяц этот код будет читать незнакомый разработчик (или ты сам, но забывший всё).

---

## 🧱 ОБЩИЕ ПРАВИЛА

### Отступы
- Используй **2 пробела** для одного уровня вложенности (не табуляцию)
- В VS Code можно настроить: Tab = 2 spaces

### Точка с запятой
- Ставь `;` в конце каждого выражения (кроме блоков `{}`)
- Это помогает избежать неявных ошибок

```javascript
// Хорошо
let score = 0;
let name = "Pixel";

// Плохо (работает, но может привести к багам)
let score = 0
let name = "Pixel"
```

Длина строки

· Старайся не писать строки длиннее 80 символов
· Длинные строки разбивай на несколько

```javascript
// Хорошо
if (veryLongVariableName === anotherVeryLongVariableName &&
    someOtherCondition === true) {
    doSomething();
}
```

## 📛 ИМЕНОВАНИЯ

Переменные и функции — camelCase

```javascript
let mouseSpeed = 5;
let isGameRunning = true;

function moveMouse() { ... }
function collectCheese() { ... }
```

Классы и конструкторы — PascalCase

```javascript
class Player { ... }
class LevelLoader { ... }
```

Константы (неизменяемые значения) — UPPER_SNAKE_CASE

```javascript
const MAX_SPEED = 10;
const DEFAULT_SCORE = 0;
```

Имена файлов

· Только латиница, без пробелов
· Для JS: game.js, levelLoader.js
· Для CSS: style.css
· Для JSON: level1.json

## 💬 КОММЕНТАРИИ

Когда комментировать

· Сложные алгоритмы
· «Магические числа» (почему именно 5, а не 10)
· Баги или временные решения

Формат комментариев

```javascript
// Подсчёт очков: +10 за сыр, +20 за супер-сыр
function calculatePoints(itemType) { ... }

/* 
 * Обработка столкновений работает через AABB-алгоритм
 * Пока что без оптимизации, на 3 уровнях держится
 */
function checkCollisions() { ... }
```

Чего избегать

```javascript
// Плохо — очевидный комментарий
i++; // увеличивает i на 1

// Плохо — закомментированный старый код (удаляй)
// let oldVariable = 5;
```

## 🧩 ОРГАНИЗАЦИЯ КОДА

Порядок в файлах

1. Импорты (если есть)
2. Константы (UPPER_CASE)
3. Переменные состояния (let, const)
4. Функции
5. Инициализация и запуск

```javascript
// 1. Константы
const MAX_MOUSE_SPEED = 8;

// 2. Переменные
let score = 0;
let level = 1;

// 3. Функции
function moveMouse() { ... }
function checkCollisions() { ... }

// 4. Запуск
function init() {
    moveMouse();
}
```

Одна функция — одна задача

```javascript
// Хорошо — функция делает одно
function updateScore(points) {
    score += points;
    updateScoreDisplay();
}

// Плохо — функция делает всё
function doEverything() {
    // передвижение, проверка, отрисовка, звуки...
}
```

## 🎨 ФОРМАТИРОВАНИЕ

Фигурные скобки — на той же строке

```javascript
// Хорошо
if (condition) {
    doSomething();
}

// Плохо
if (condition)
{
    doSomething();
}
```

Пробелы вокруг операторов

```javascript
// Хорошо
let sum = a + b;
if (x === y) { ... }

// Плохо
let sum=a+b;
if (x===y){...}
```

## 🚫 ЧЕГО ИЗБЕГАТЬ

· Глобальные переменные — не плоди window.myVar
· магические числа — выноси в константы
· дублирование кода — если повторяется 3 раза, вынеси в функцию
· глубокую вложенность — больше 3 уровней {} уже сложно читать

```javascript
// Плохо — глубоко вложено
if (a) {
    if (b) {
        if (c) {
            if (d) {
                doSomething();
            }
        }
    }
}

// Хорошо — выход раньше
if (!a) return;
if (!b) return;
if (!c) return;
if (d) doSomething();
```

## 🐱 АЙКОТ ГОВОРИТ

«Стиль кода — это вежливость. Ты оставляешь после себя код, который не стыдно показать другим. И себе через месяц».

«Главное правило: работает лучше, чем красивый, но неработающий. Сначала сделай так, чтобы работало. Потом сделай красиво».
